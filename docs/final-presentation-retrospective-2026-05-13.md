## 발표에서 강조한 구현

SchoolBridge의 번역 파트는 "NLLB로 번역했다"에서 멈추지 않고, 학교 안내문에서 실제 행동 오류로 이어질 수 있는 값을 보호하는 구조로 설계했다.

| 구성 | 역할 |
| --- | --- |
| Slot Protection | 날짜, 금액, URL, 전화번호를 `__SLOT0__` 형태로 격리한 뒤 번역 후 복원 |
| Glossary Injection | 학교 도메인 용어를 권장 번역어로 고정해 NLLB 오역을 줄임 |
| Template-based translation | 준비물, 제출물, 납부 등 반복 문장은 구조화 후 템플릿으로 번역 |
| Unknown-term loop | glossary에 없는 용어를 수집해 다음 번역 품질 개선 후보로 남김 |
| TTS 연결 | 번역 결과를 베트남어 음성 안내까지 이어지게 구성 |

## 받은 피드백

좋았던 점:

- 슬롯, 마스킹, glossary injection, template 기반 번역 구조가 실제 서비스 문제를 잘 잡았다.
- 단순 모델 호출이 아니라 NLLB의 약점을 알고 보완한 구현으로 보였다.
- 구현 결과를 수치로 설명해 설득력이 있었다.

아쉬운 점:

- 학습과 출력 처리 흐름을 한 장 더 자세히 보여주면 더 좋다.
- 발표자료 일부에 오타 또는 잘려 보이는 텍스트가 있었다.
- p18 흐름도 2번 박스는 `미등록 용어 감지`가 끝까지 보이도록 수정할 필요가 있다.
