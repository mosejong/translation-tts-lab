# 번역 품질평가 개선 공유 요약

작업일: 2026-04-28  
작업자: 모세종 / Codex 보조  
목적: Gemini 평가 100점 몰림과 NLLB 번역 잘림 문제 개선

---

## 한 줄 요약

기존 평가는 “사전 용어가 들어갔는지” 중심이라 점수가 과하게 높게 나왔고, 긴 통신문은 NLLB `max_length=256` 제한 때문에 중간에 잘렸습니다.  
이번 수정으로 **청크 번역 + 현지 상용 표현 평가 + 한국어 역번역(Round-trip) 검사**를 추가했습니다.

---

## 해결한 문제

| 문제 | 기존 상태 | 개선 내용 | 결과 |
|---|---|---|---|
| 번역 중간 잘림 | 긴 통신문이 `max_length=256`에서 잘림 | 문장 단위 청크 번역 후 결과 합치기 | 긴 문서도 끝까지 번역 가능 |
| Gemini 100점 몰림 | 용어사전 적용 후 대부분 100점 | 후처리 점수 상한 및 엄격 기준 추가 | 100점 몰림 제거 |
| 단순 용어 포함 평가 | 사전 단어 포함 여부 중심 | 실제 현지 학부모 안내문에서 자주 쓰는 표현인지 평가 | 자연스러운 표현까지 반영 |
| 의미 왜곡 탐지 부족 | 번역문만 보고 평가 | 번역문을 다시 한국어로 의미 역번역해 원문과 비교 | 누락/왜곡/환각 탐지 가능 |

---

## 새 평가 기준

| 평가 축 | 확인 내용 |
|---|---|
| 정확도 | 원문 핵심 행동, 대상, 일정, 금액, 준비물이 유지되는가 |
| 현지 자연스러움 | 해당 언어권 학부모 안내문에서 실제로 쓰는 표현인가 |
| 학교 문맥 | 직역이 아니라 학교/유치원 상황에 맞는 단어인가 |
| 정보 보존 | 날짜, 시간, 금액, 제출처, 마감일, 주의사항이 빠지지 않았는가 |
| Round-trip | 다시 한국어로 역번역했을 때 원문 의미가 살아 있는가 |

---

## 재측정 결과

### A/B 번역 속도 비교

| 항목 | 결과 |
|---|---:|
| 대상 | 베트남어 18개 공지 |
| 입력 단축 | -30.1% |
| 속도향상 | x1.84 |
| 출력 위치 | `translation/outputs/ab_compare/vi/` |

### A/B 번역 품질 평가

| 항목 | 기존 평가 | 새 평가 |
|---|---:|---:|
| A: 원문 전체 번역 | 33.3점 | 45.8점 |
| B: TODO만 번역 | 45.1점 | 50.1점 |
| B 우세 폭 | +11.8점 | +4.3점 |

해석: 새 평가는 훨씬 엄격합니다. B가 여전히 평균적으로 높지만, 문서에 따라 원문 전체 맥락이 필요한 경우 A가 더 좋은 케이스도 확인됐습니다.

### 용어사전 전/후 품질 평가

| 항목 | 기존 평가 | 새 평가 |
|---|---:|---:|
| NLLB 원번역 평균 | 33.9점 | 39.0점 |
| 용어사전 보정 후 평균 | 100.0점 | 89.6점 |
| 향상폭 | +66.1점 | +50.6점 |

해석: 기존 100점은 평가 기준이 후했던 1차 결과였습니다. 새 기준에서는 현지 자연스러움과 역번역 의미 보존까지 봐서 더 현실적인 점수로 조정됐습니다.

## Round-trip 검사 예시: N02 초등안심벨

| 항목 | 점수 | 역번역으로 잡힌 문제 |
|---|---:|---|
| A: 원문 전체 | 45점 | 본문 후반 누락, 안심벨 오역, 학생 대상 왜곡 |
| B: TODO만 | 72점 | `장난 사용 주의`가 `가구 사용 주의`로 왜곡, `어깨끈`이 `손목끈`으로 왜곡 |

의미: 겉보기에는 베트남어 문장처럼 보여도, 다시 한국어로 돌려보면 실제 안내 의미가 틀어진 부분을 잡을 수 있습니다.

---

## 발표/미팅 때 말할 포인트

1. 단순 번역 속도만 본 것이 아니라, 품질 평가 기준도 강화했습니다.
2. 기존 “100점” 결과는 1차 평가였고, 지금은 더 엄격한 기준으로 재평가했습니다.
3. 현지 언어에서 실제로 쓰는 표현인지까지 평가하도록 바꿨습니다.
4. 번역문을 다시 한국어로 역번역해 의미 누락/왜곡/환각을 잡는 검사 로직을 추가했습니다.
5. 결과적으로 TODO 추출은 평균적으로 속도와 품질 모두 이점이 있지만, 문서 유형별로 효과 차이가 있음을 근거로 확인했습니다.
