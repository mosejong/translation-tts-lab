# MVP Briefing

## MVP 한 줄 설명

학교/유치원 가정통신문을 입력하면 핵심 내용을 쉬운 한국어로 정리하고, 베트남어 번역과 학교 용어 검수를 거쳐 음성 안내까지 생성하는 E2E 데모입니다.

## 전체 파이프라인

```text
가정통신문 원문 입력
-> 베이스라인 분석/요약
-> 쉬운 한국어 변환
-> Hugging Face NLLB 기반 베트남어 번역
-> term_glossary.csv 기반 학교 용어 검수
-> 베트남어 TTS 음성 출력
```

현재 데모 산출물은 `outputs/mvp/` 아래에 저장됩니다.

| File | Description |
| --- | --- |
| `01_input_notice.txt` | 입력 가정통신문 원문 |
| `02_baseline_result.json` | 카테고리, 키워드, 사전 감지 결과 |
| `03_easy_ko.txt` | 쉬운 한국어 문장 |
| `04_vi_translation.txt` | 베트남어 번역 결과 |
| `05_glossary_check.csv` | 학교 용어 검수 결과 |
| `06_tts_output.wav` | 베트남어 음성 출력 |
| `mvp_result.csv` | 전체 결과 통합 파일 |

## Hugging Face 번역 모델 사용 이유

현재는 번역 모델을 처음부터 새로 학습시키지 않고, Hugging Face의 사전학습 다국어 번역 모델인 `facebook/nllb-200-distilled-600M`을 베이스라인으로 사용합니다.

이 방식은 짧은 MVP 기간 안에 실제 입력에서 번역 결과를 만들 수 있고, 이후 데이터가 쌓이면 파인튜닝 또는 후처리 개선으로 확장할 수 있습니다.

## term_glossary.csv 역할

일반 번역 모델은 학교/유치원 가정통신문 문맥에 약합니다. 예를 들어 `체험학습비`, `귀가 동의`, `참가 여부`, `도화지`, `원복` 같은 표현은 일반 문맥과 다르게 번역될 수 있습니다.

이를 보완하기 위해 약 100여 개의 학교 특화 용어를 `translation/term_glossary.csv`에 정리했습니다.

번역 단계에서는 입력문에서 사전 용어를 감지하고, 번역문에 권장 베트남어 표현이 반영되었는지 확인합니다.

품질 라벨 기준:

| Label | Meaning |
| --- | --- |
| `ok` | 입력문에 나온 사전 용어가 번역문에도 권장 베트남어로 반영됨 |
| `missing_term` | 입력문에 사전 용어가 있었지만 번역문에 권장어가 없음 |
| `unchecked` | 사전 용어가 없어 자동 판단 불가 |
| `review_needed` | 사람이 확인해야 하는 항목 |

## Gemini API의 역할

Gemini API는 최종 번역기가 아닙니다.

추후 새 가정통신문에서 미등록 용어가 발견되면, Gemini API를 사용해 베트남어 초안을 추천받을 수 있습니다. 다만 추천 결과는 바로 사전에 등록하지 않고, 사람이 검수한 뒤 `term_glossary.csv`에 추가합니다.

```text
미등록 용어 후보 추출
-> Gemini가 베트남어 초안 추천
-> 사람이 검수
-> 승인된 표현만 term_glossary.csv에 추가
```

## 베트남어 한정 MVP의 의미

현재 MVP는 베트남어를 대상으로 검증합니다. 하지만 구조는 특정 언어에만 묶여 있지 않습니다.

언어별 용어 사전과 TTS 모델을 추가하면 중국어, 영어, 러시아어, 몽골어 등으로 확장할 수 있습니다. 따라서 현재 베트남어 MVP는 다문화 가정 및 국제학교 안내문 번역 시스템을 검증하기 위한 첫 번째 언어 버전입니다.

## 향후 다국어 확장 방향

1. 언어별 `term_glossary` 확장
2. 언어별 TTS 모델 또는 API 연결
3. 검수 완료 번역 데이터 누적
4. NLLB 번역 결과와 검수 데이터를 비교해 품질 개선
5. 충분한 데이터가 쌓이면 파인튜닝 검토

## 현재 한계점

- NLLB 원본 번역은 학교 문맥 용어를 안정적으로 처리하지 못합니다.
- `term_glossary.csv`는 사람이 계속 확장해야 합니다.
- 현재 사전 검수는 문자열 포함 여부 기반이라 표현 변형을 완벽히 잡지는 못합니다.
- TTS는 번역문 품질이 충분히 좋을 때만 실제 서비스 품질을 보장할 수 있습니다.
- 현재 데모는 베트남어 중심이며, 다른 언어는 추가 사전과 음성 모델 검증이 필요합니다.
