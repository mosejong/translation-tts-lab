## 📌 담당 파트 개요

모델3는 모델1(추출)·모델2(분류)가 만든 결과를 학부모가 실제로 이해할 수 있는 형태로 전달하는 최종 단계다.

| 역할 | 설명 |
|---|---|
| 데이터셋 구축 | 가정통신문 라벨링 샘플 v1~v3 제작 |
| NLLB 번역 | facebook/nllb-200-distilled-600M 기반 한→베 번역 |
| 용어사전 보정 | 학교 도메인 핵심 용어 누락 보완 |
| Gemini 보조 | 신규 용어 설명 초안 생성 (번역 메인 아님) |
| Edge-TTS | vi-VN-HoaiMyNeural 음성 생성 |
| Android 연결 안정화 | ReadTimeout, BASE_URL, fallback 처리 |

### 🐛 핵심 버그 — ReadTimeout 10초

| 항목 | 내용 |
|---|---|
| 증상 | 분석 버튼 누르면 항상 고정 fallback 데이터 표시 |
| 원인 | `conn.setReadTimeout(10000)` — NLLB 번역이 30초 이상 걸려 timeout |
| 확인 | Docker 로그에 Android에서 analyze 요청 자체가 도달하지 않음 |
| 수정 | `conn.setReadTimeout(60000)` 으로 변경 |

### 🛠️ 추가 수정

- `BASE_URL` 개인 IP → placeholder `http://192.168.x.x:8000` 으로 변경
- docs: E2E 파이프라인 연결 완료 기준으로 문서 최신화
- docs: README 진도 현황 E2E 완료 반영
- docs: E2E 테스트 이슈 3건 troubleshooting 추가

### 💭 회고

30초 넘는 NLLB 추론을 10초 timeout이 막고 있었다. Docker 로그로 "Android에서 요청 자체가 안 왔다"는 걸 확인하기 전까지 원인을 찾는 데 시간이 걸렸다. 로그 먼저 보는 습관이 중요하다는 걸 다시 배웠다.

- `google-genai` SDK 사용 (`google.generativeai` deprecated 대응)

- retry 개선: API 응답의 "retry in Xs" 파싱 → 정확한 대기 + 일일 quota 소진 감지 시 즉시 스킵

- must_idx: 용어사전 단어 포함 문장 인덱스 추출
- other_idx: 나머지 문장 인덱스
- selected: must_idx 우선 채우고, 빈 슬롯만 other_idx로 채움 (정렬 유지)

### 💭 회고

강사님 피드백의 핵심은 "돌아가는 것"과 "증명된 것"은 다르다는 것이다. NLLB가 Papago보다 나은 게 아니라, Papago도 실패하는 통화 오번역을 우리가 후처리로 잡았다는 것이 우리 강점이다. 이걸 수치와 사례로 정리하는 게 다음 단계의 핵심이다.

오늘 하루에 파이프라인 전체를 vi 전용 → 8개 언어 범용으로 바꾸면서, Gemini quota 관리와 배치 설계가 비용 최적화의 핵심임을 실감했다. 144개 × 8개 언어 = 1,152번 호출을 8번으로 줄인 설계가 무료 한도 안에서 운영 가능하게 해준다.
